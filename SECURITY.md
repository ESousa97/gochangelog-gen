# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| Latest release | Yes |
| Older releases | No |

## Reporting a Vulnerability

If you discover a security vulnerability in gochangelog-gen, please report it responsibly.

**Do not open a public issue.** Instead:

1. Email: **security@enoquesousa.dev** (or open a private security advisory on GitHub)
2. Include a clear description of the vulnerability
3. Provide steps to reproduce if possible
4. Allow reasonable time for a fix before public disclosure

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Fix release**: As soon as practical after confirmation

## Scope

This policy covers the gochangelog-gen CLI tool and its dependencies. It includes:

- Code execution vulnerabilities
- Dependency vulnerabilities (CVEs)
- Credential exposure risks
- Path traversal or injection issues

Thank you for helping keep gochangelog-gen secure.
